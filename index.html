给定一个字符串 s ，请你找出其中不含有重复字符的最长子串的长度。

@3. 无重复字符的最长子串leetcode-cn.com/problems/longest-substring-without-repeating-characters

解法一

暴力解法，用两个循环穷举所有子串，然后再用一个函数判断该子串中有没有重复的字符。

时间复杂度：两个循环，加上判断子串满足不满足条件的函数中的循环，O(n^3)。
空间复杂度：使用了一个 set，判断子串中有没有重复的字符。由于 set 中没有重复的字符，所以最⻓就是整个字符集，假设字符集的大小为 m，那么 set 最⻓就是 m。另一方面，如果字符串的⻓度小于 m，是 n。那么set最⻓也就是 n 了。综上，空间复杂度为 O(min(m, n))。

解法二

解法一太过暴力使得时间复杂度过高。分析一下有以下几点可以优化：

  1. 当 i 取 0 时，依次判断了字符串 str[0,1), str[0,2), str[0,3)……中有没有重复的字符，这样就进行了很多重复的判断，在判断 str[0,2) 中没有重复字符后，只需要判断 str[2] 在不在 str[0,2) 中即可；而如果 str[0,2) 中有重复字符，则包含 str[0,2) 的字符串如 str[0,3), str[0,4)……中也有重复字符。
  2. 判断一个字符在不在字符串中，我们需要遍历整个字符串，时间复杂度为 O(n)。而使用 Hash 来存储字符串，查找的时间复杂度就降为了 O(1)。
    此时，可以理解为一个滑动窗口在遍历整个字符串。

时间复杂度：在最坏的情况下，while 循环中的语句会执行 2n 次，例如 abcdefgg，开始的时候 rk 一直后移，直到到达第二个 g 的时候固定不变 ，然后 i 开始一直后移直到 n ，所以总共执行了 2n 次，时间复杂度为 O(n)。
空间复杂度：和解法一类似，需要一个 Hash 保存子串，所以是 O(min(m, n))。

给你两个非空的链表，表示两个非负的整数。它们每位数字都是按照逆序的方式存储的，并且每个节点只能存储一位数字。

请你将两个数相加，并以相同形式返回一个表示和的链表。

你可以假设除了数字 0 之外，这两个数都不会以 0 开头。

@2.两数相加leetcode-cn.com/problems/add-two-numbers

思路

每一位相加可能会产生最大为 1 的进位，我们用 carry 表示。
创建一个头节点为 Head 的链表来储存结果。
对两个要相加的链表进行遍历相加，加 1 位更新一次 sum ，用 sum 的十位更新 carry，个位存储到结果链表。

空间复杂度：O(max(m, n))，m 和 n 代表 l1 和 l2 的⻓度。
而其实新的 List 最大⻓度是 O(max(m, n))+1，因为我们的 Head 没有存储值。

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出和为目标值 target的那两个整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。

你可以按任意顺序返回答案。

@1.两数之和leetcode-cn.com/problems/two-sum

解法一

两重循环，遍历所有情况看相加是否等于目标和，如果符合直接输出。

时间复杂度：两层 for 循环，O(n2)；
空间复杂度：O(1)。

解法二

在解法一中的第二个 for 循环：

遍历所有元素，寻找其中与 num[i] 之和为 target 的元素，时间复杂度为 O(n)。
而使用 Hash table，可以不用遍历所有元素就找出 num[j]。

我们将数组的每个元素保存为 hash 的 key，下标保存为 hash 的 value。
这样只需判断 num[j] 在不在 Hash 的 key 里就可以了，而此时的时间复杂度仅为 O(1)。
需要注意的地方是，还需判断找到的元素不是当前元素，因为一个元素只能用一次。

时间复杂度：比解法一少了一个 for 循环，降为 O(n)；
空间复杂度：以空间换时间， 开辟了一个 Hash table，空间复杂度变为 O(n)。
